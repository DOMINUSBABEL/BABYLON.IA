## 2024-06-25 - Fix Path Traversal in Server Path Validation
**Vulnerability:** A Path Traversal vulnerability existed in the `resolveAndValidatePath` helper inside `src/server.js`. The validation logic used `!fullPath.startsWith(path.resolve(basePath))` to ensure paths remained within boundaries. However, this is a flawed implementation because string checking allows an attacker to specify a relative path that breaks out into sibling directories with a matching prefix. For instance, if `basePath` is `/app/workspace`, an attacker could request `../workspace2/file.txt`, which resolves to `/app/workspace2/file.txt`. This path string still *starts with* `/app/workspace`, bypassing the security check.
**Learning:** `startsWith()` applied to simple path strings is inherently insecure for boundary validations due to prefix overlap (e.g. `workspace` vs `workspace-secret`).
**Prevention:** Proper boundary validation requires exact path match checks (`fullPath === resolvedBase`) or ensuring that the string validation explicitly incorporates the path separator (`fullPath.startsWith(resolvedBase + path.sep)`) so that it genuinely refers to a child directory and not just a sibling directory sharing a name prefix.
